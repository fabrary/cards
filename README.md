# Flesh and Blood cards

A monorepo of **Flesh and Blood** card data and tooling, published as three packages and managed with
[lerna](https://lerna.js.org/) + [nx](https://nx.dev/) + npm workspaces. All three share a single
version (see `lerna.json`).

## Packages

| Package                                      | What it is                                                                       |
| -------------------------------------------- | -------------------------------------------------------------------------------- |
| [`@flesh-and-blood/types`](packages/types)   | Enums, interfaces, and helper functions (`getDefaultPrinting`, `getPrint`, etc.) |
| [`@flesh-and-blood/cards`](packages/cards)   | Every Flesh and Blood card as a single `cards: Card[]` export                    |
| [`@flesh-and-blood/search`](packages/search) | A [fuse.js](https://www.fusejs.io/)-based search engine                          |

All packages publish **dual ESM + CJS** for maximum compatibility.

## Quick start

Install the package(s) you need:

```bash
npm i @flesh-and-blood/cards @flesh-and-blood/types   # card data + types
npm i @flesh-and-blood/search @flesh-and-blood/types  # search engine (types is a peer dependency)
```

```ts
import { cards } from "@flesh-and-blood/cards";
import { Hero } from "@flesh-and-blood/types";

const rhinarSpecializations = cards.filter((card) =>
  card.specializations?.includes(Hero.Rhinar),
);
```

See each package's README for full usage.

## Card data

Card data is sourced from the amazing
[the-fab-cube/flesh-and-blood-cards](https://github.com/the-fab-cube/flesh-and-blood-cards) project by
[Tyler Luce](https://github.com/luceleaftea) — all credit to him; any errors are probably mine 😅. It is
generated by a transform pipeline in the `cards` package — see
[`packages/cards/README.md`](packages/cards) for the full pipeline.

## Working in this repo

`.nvmrc` is `lts/*` — run `nvm use` if your Node version looks off.

Commands run across all packages via lerna (build order is **types → cards → search**, handled by nx):

| Command         | What it does                                           |
| --------------- | ------------------------------------------------------ |
| `npm run build` | Build every package (`lerna run build`)                |
| `npm test`      | Run every package's tests                              |
| `npm run full`  | Regenerate card data and rebuild everything end to end |
